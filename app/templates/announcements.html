Announcements

{{SITE_NAME}} is a website created to aid UofC students in their program course planning. The Course Browser allows users to quickly browse through the university's full course catalog, with intuitive filter and sorting options. By clicking on any course in the Course Browser, the course's view page opens with all the essential information for students. With an optional account, users can save courses with custom tags and add them to a virtual Course Planner. In the Course Planner, students can arrange courses in all the terms they have been enrolled in, and are planning on enrolling in the future. Grades can be added to courses inside terms that have concluded, and the GPA for the term will be automatically calculated.
{{SITE_NAME}} also offers a public Web API to easily retrieve Grade, Term, Faculty, Subject and Course data within your own application. The available endpoints are listed in the API documentation.
We plan to regularly add many more features that will keep on improving students' experience making the best choices for their career plan.

Why?

{{SITE_NAME}} was inspired by the lack of ability in the university's public and official medium for browsing through courses. Design-wise, according to the Wayback Machine, the site has been practically left untouched since 2009! More than a decade later, it's clear that a website every student is meant to use requires a modern refresh. In terms of functionality, the website makes it hard to sort through the selection of courses available without any filtering functionality. Even with a good course browsing experience, it is expected that students find their own resources to organize their selected and desired courses. {{SITE_NAME}} aims to provide a uniform experience for general course planning that adapts to every student, with more features to come.

How?

The back-end of this website was developed in Python using the Flask micro web framework. To store all the site data, including user data, a relational MySQL database is used and connected to Flask using the SQLAlchemy library. User-sensitive data, like passwords, are stored as a hash using the Bcrypt hashing algorithm. Front-end-wise, the layout and design of this website was created using the Bootstrap framework.

Who?
